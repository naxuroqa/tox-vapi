Tox

Object Hierarchy:

Description:

[ Compact ]
public class Tox

The Tox instance type. All the state associated with a connection is held within the instance. Multiple instances can exist and operate concurrently. The maximum number of Tox instances that can exist on a single network device is limited. Note that this is not just a per- process limit, since the limiting factor is the number of usable ports on a device.

  • public delegate void FileRecvCallback (Tox self, uint32 friend_number, uint32 file_number, uint32 kind, uint64 file_size, uint8[] filename, void* user_data)
    The client should acquire resources to be associated with the file transfer. Incoming file transfers start in the PAUSED state. After this callback returns, a transfer can be rejected by sending a TOX_FILE_CONTROL_CANCEL control command before any other control commands. It can be accepted by sending TOX_FILE_CONTROL_RESUME.
